# Evolution Roadmap

Meridian should evolve from a prompt-assisted workflow into a reusable analysis
operating system.

## Stage 0: Prompted Discipline

Current state:

- one general Agent skill
- basic analysis contract and state templates
- one consistency checker
- one cross-border domain pack

Goal:

- prove that explicit state and quality gates reduce drift in real analysis.

## Stage 1: Protocolized Harness

Next step:

- stabilize core protocols
- define required run artifacts
- define domain pack manifests
- separate prompt adapters from core logic

Success signal:

- a new domain pack can be created without changing the core skill from scratch.

## Stage 2: Repeatable Runs

Add:

- run initialization
- source log templates
- claim ledger templates
- review result templates
- extraction records

Success signal:

- two analysts or two agents can run the same task and produce comparable
  artifacts.

## Stage 3: Evaluation Loop

Add:

- human review records
- benchmark cases
- failure mode taxonomy
- before/after comparisons
- historical prediction checks where possible

Success signal:

- the system can explain why one analysis output is more reliable than another.

## Stage 4: Domain Pack Library

Add domain packs such as:

- cross-border market opportunity
- industry research
- company strategy
- investment memo
- product opportunity analysis
- competitive intelligence

Success signal:

- domain packs share the same core lifecycle while keeping domain-specific
  metrics and evaluation rules.

## Stage 5: Workbench

Only after the protocol stabilizes, build a UI or hosted workbench around:

- contract editing
- source and claim review
- report generation
- quality gate status
- domain asset extraction

Success signal:

- the interface makes the harness easier to follow without hiding the state.
