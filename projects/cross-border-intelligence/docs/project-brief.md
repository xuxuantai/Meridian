# Project Brief

## Name

Cross-border Decision Intelligence Engine

## Vision

Structure the working method of strong business analysts, then scale that method
through general-purpose AI agents.

The long-term system should help companies answer questions such as:

- Which market should we enter next?
- Which countries are showing demand growth?
- Where is competition still manageable?
- Which markets match our supply chain?
- What product test should we run next?

## First Wedge

The first validation domain is cross-border market opportunity analysis:

```text
country x product x industry cluster
```

This wedge is attractive because public data is rich, the decision value is
clear, and research outputs are naturally publishable as professional content.

## What This Project Is Not

- Not a generic product selection tool.
- Not a raw data query tool.
- Not a marketplace for prompts, skills, or plugins.
- Not a new general agent runtime.

## Product Path

1. Content-led acquisition through public research.
2. Paid custom decision services.
3. Market intelligence subscription.
4. Enterprise AI research workspace.

## Technical Strategy

Use general agents such as Codex, OpenCode, Claude Code, or OpenHands as the
execution layer. Build proprietary value in:

- analysis skills
- domain knowledge
- metric definitions
- evaluation mechanisms
- historical cases
- human feedback loops

## V0 Deliverables

- Repository structure.
- Analysis Workspace CLI.
- First cross-border analysis task template.
- YAML schemas for skill, metric, task, and evaluation assets.
