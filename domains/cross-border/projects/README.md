# Projects

Each folder in `projects/` represents one analysis engagement or public research
piece.

Suggested structure:

```text
projects/2026-01-outdoor-lighting-gcc/
  task.yaml
  notes.md
  sources.yaml
  analysis/
  report/
  review/
  extracted-assets/
```

Keep private customer data in `private/` and raw downloaded data in `raw/`.
Both are ignored by git by default.
